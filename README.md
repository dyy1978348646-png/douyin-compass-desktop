# 抖音罗盘数据抓取器

一个面向 macOS / Windows 的桌面工具，用来自动打开抖音罗盘、进入指定数据页面、导出数据，并保存为本地文件与 SQLite 数据库。

这个项目支持两种使用方式：

1. 普通用户：直接下载已经打包好的桌面应用
2. 开发者或高级用户：下载源码后自己运行或自己打包

如果你只是想“安装后直接双击使用”，请优先看下面的“普通用户安装方式”。

---

## 1. 项目能做什么

当前版本主要包含这些功能：

- 打开抖音罗盘并使用浏览器持久化登录
- 支持达人入口和店铺入口
- 自动导航到对应的数据页面
- 自动导出 Excel 数据
- 自动保存 CSV
- 自动写入 SQLite 数据库
- 图形界面操作，不需要命令行长期值守
- 支持近期七天、近一天、自定义日期抓取
- 支持“一键导出最近一次抓取数据”
- 支持定时执行

## 2. 普通用户安装方式

如果你不是开发者，不建议直接下载源码。

正确做法是：

1. 打开本项目的 GitHub Releases 页面
2. 根据你的电脑系统下载对应压缩包
3. 解压后直接运行

### 2.1 下载哪个文件

通常会有两类发布包：

- macOS 用户：下载 `douyin-compass-scraper-macos.dmg`
- Windows 用户：下载 `douyin-compass-scraper-windows.zip`

### 2.2 macOS 用户怎么安装

1. 下载 `douyin-compass-scraper-macos.dmg`
2. 双击打开 `.dmg`
3. 在安装窗口中把 `抖音罗盘抓取器.app` 拖到 `Applications`
4. 打开 `Applications`
5. 双击启动 `抖音罗盘抓取器`

### 2.3 macOS 第一次打不开怎么办

这是最常见的新手问题。

因为这个应用目前没有做 Apple 开发者签名，macOS 可能会拦截首次打开。

你可以按下面方式处理：

#### 方法 A：右键打开

1. 找到 `抖音罗盘抓取器.app`
2. 右键
3. 选择“打开”
4. 再次确认打开

#### 方法 B：在终端里去除隔离标记

如果还是打不开，可以执行：

```bash
xattr -cr "/Applications/抖音罗盘抓取器.app"
```

如果你的应用不在 `Applications`，把路径改成实际路径即可。

### 2.4 Windows 用户怎么安装

1. 下载 `douyin-compass-scraper-windows.zip`
2. 解压
3. 打开解压后的文件夹
4. 运行 `抖音罗盘抓取器.exe`

建议不要直接在压缩包里双击运行，一定要先完整解压。

### 2.5 第一次启动为什么会比较慢

从当前版本开始，发布包已经内置 Playwright 的 Chromium 浏览器组件。

也就是说：

- 首次启动不再需要联网下载浏览器
- Windows 下不会再弹出 Chromium 安装用的空白 CMD 窗口
- 首次只需要等待程序正常解压/初始化即可

浏览器组件安装位置：

- 打包应用内部自带一份 Chromium
- 只有当内置浏览器缺失时，程序才会回退到用户目录重新安装

## 3. 使用前你需要知道的事

这个工具不是直接调用抖音官方开放 API，而是通过自动化浏览器操作罗盘网页。

所以你在第一次使用时，会经历这些步骤：

1. 程序启动浏览器
2. 你手动登录抖音罗盘
3. 如有需要，你手动选择账号
4. 程序记住你的登录状态
5. 后续运行通常不用重复登录

这意味着：

- 首次使用时要有耐心
- 浏览器不要随便在任务执行中被你强行关闭
- 某些页面结构变动后，抓取流程可能需要调整代码

## 4. 图形界面怎么用

打开软件后，一般会看到三个区域：

- 账号配置
- 调度设置
- 运行日志

### 4.1 账号配置

你可以在这里：

- 选择入口类型
- 选择抓取日期范围
- 输入自定义日期
- 填写罗盘地址
- 查看当前登录状态
- 切换账号
- 清除浏览器数据重新登录

入口类型说明：

- 达人入口：直播 → 直播复盘
- 店铺入口：直播 → 实时直播数据

日期范围说明：

- `近期七天`：默认选项，适合日常巡检
- `近一天`：适合快速抓最近一天数据
- `自定义日期`：支持输入如 `2026-03-28`、`2026年3月28日`、`3月28日`、`昨天`

### 4.2 调度设置

你可以在这里：

- 设置每天自动执行时间
- 启动调度
- 停止调度

### 4.3 运行日志

这里会显示程序每一步正在做什么，例如：

- 是否已打开罗盘
- 是否检测到登录
- 是否进入仪表盘
- 是否成功导出文件
- 文件保存到了哪里

### 4.4 底部按钮

常用按钮包括：

- `立即执行一次`
- `取消任务`
- `导出最近一次数据`
- `最小化到托盘`

其中 `导出最近一次数据` 的作用是：

你不需要再去日志里找文件路径，直接在界面里选择一个保存位置，就能把最近一次抓取结果导出出来。

## 5. 抓取结果保存在哪里

程序会把不同类型的数据保存到用户目录中。

### 5.1 macOS

目录位置：

```text
~/Library/Application Support/抖音罗盘抓取器/
```

常见内容：

- `config.json`：配置文件
- `logs/`：运行日志
- `downloads/`：浏览器导出的原始 Excel 文件
- `data/`：程序整理后的 CSV 与 SQLite 数据
- `ms-playwright/`：仅在内置浏览器缺失、程序回退到用户目录安装时出现
- `browser_profile/`：浏览器登录状态和持久化数据

### 5.2 Windows

目录位置：

```text
%APPDATA%\抖音罗盘抓取器\
```

内容类型与 macOS 基本一致。

## 7. 常见问题

### 7.1 为什么我下载了源码却不能直接双击运行

因为源码不是打包后的桌面应用。

源码需要你自己安装：

- Python
- 项目依赖
- 打包工具

然后要么用命令运行，要么自己打包。

所以普通用户应下载 Release 中的打包版本，而不是源码压缩包。

### 7.2 为什么第一次运行会要求联网

正常发布包不需要。

如果你看到首次运行仍然要求联网，通常说明：

- 你运行的是源码，不是打包版本
- 打包时没有先执行浏览器预装步骤
- 打包环境中的 Playwright 浏览器资源没有正确打进应用

### 7.3 为什么登录一次后下次不用重新登录

因为程序使用持久化浏览器配置，会保存你的登录状态。

### 7.4 为什么某天突然不能抓了

常见原因有：

- 抖音罗盘页面结构变化
- 登录状态失效
- 目标账号需要重新选择
- 网络异常

这类问题一般先看“运行日志”页。

### 7.5 导出的文件在哪里找

优先直接使用界面里的 `导出最近一次数据`。

如果你要手动找，去用户数据目录里的：

- `downloads/` 看原始 Excel
- `data/` 看整理后的 CSV 和数据库

如果你是开发者，或者你确实想自己调试源码，可以按下面做。

### 8.2 安装依赖


- `data/`

## 9. 开发者方式：自己打包

当前打包流程会先在干净虚拟环境里安装依赖，再优先复用本机已有的 Playwright 浏览器缓存；如果缓存缺失，才会在线安装 Chromium。最后通过 `douyin_compass.spec` 一起打包进去。

这样做的目的有两个：

- 普通用户首次运行时不再下载浏览器
- Windows 打包版不会再弹出 Chromium 安装相关的空白控制台窗口

另外，`build_mac.sh` 和 `build.bat` 现在都会先创建一个干净的打包虚拟环境，再在该环境里安装依赖并执行 PyInstaller。

这样可以避免你本机全局 Python 环境里的无关包被误打进安装包。
同时，如果你本机已经装过 Playwright Chromium，构建脚本会直接复用缓存，避免每次完整重下浏览器。

### 9.1 macOS 打包

```bash
bash build_mac.sh
```

脚本会自动创建并重建：

```text
.build-venv-macos/
```

打包完成后通常输出：

```text
dist/抖音罗盘抓取器.app
dist/抖音罗盘抓取器.dmg
```

其中：

- `dist/抖音罗盘抓取器.app` 是安装后的展开目录，体积会明显大于 `.dmg`
- `dist/抖音罗盘抓取器.dmg` 才是给普通用户分发的标准安装镜像
- 用户打开 `.dmg` 后，可以直接把应用拖到 `Applications`

### 9.2 Windows 打包

在 Windows 机器上运行：

```bat
build.bat
```

脚本会自动创建并重建：

```text
.build-venv-win\
```

打包完成后通常输出：

```text
dist\抖音罗盘抓取器_win_full.zip
```

其中：

- `dist\抖音罗盘抓取器_win_full.zip` 是给 Windows 用户分发的完整压缩包
- 用户解压后运行 `dist\抖音罗盘抓取器\抖音罗盘抓取器.exe`

注意：

- Windows 可执行文件需要在 Windows 系统上打包
- 不能直接在 macOS 上生成真正可运行的 Windows `.exe`
- 当前默认输出为 `onedir` 目录形式，而不是 `onefile`
- 这是为了避免把内置 Chromium 压成单文件后每次启动都重复解包，导致启动更慢

这更适合给非技术用户分发软件。
